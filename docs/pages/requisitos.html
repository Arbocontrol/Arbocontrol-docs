Documentos de requisitos

Requisitos funcionais do sistema [RFs]

Módulo de Registro de denúncias

Cadastro de denúncia Atualização: 11/06/2023
RF Descrição
RF01 O sistema deve permitir que qualquer usuário registre uma denúncia de possível foco de arbovirose com: CEP, Logradouro, Bairro, Número, Complemento, Referencia, telefone celular (Opcional), telefone fixo (Opcional), e-mail, imagem, local favorável de foco e observações (Opcional).
Geração de protocolosAtualização: 11/06/2023
RF Descrição
RF02 O sistema deve gerar código de protocolo a partir do número da denúncia, data e horário ao iniciar o formulário de uma denúncia.
Área de perguntas frequentesAtualização: 11/06/2023
RF Descrição
RF03 O sistema deve apresentar uma área destinada a perguntas mais frequentes sobre como registrar uma denúncia.
Área com informações de atendimentoAtualização: 11/06/2023
RF Descrição
RF04 O sistema deve apresentar uma área com informações sobre horários de atendimento, número de telefone, Email e endereço dos órgãos responsáveis pela Vigilância Epidemiológica do município.
Nota de desenvolvimentoAtualização: 31/08/2023
RF Descrição
RF05 O sistema deve disponibilizar uma nota de desenvolvimento para o público.

Módulo de administradores do sistema

Autenticação de acessoAtualização: 11/06/2023
RF Descrição
RF06 O sistema deve garantir que somente os usuários autorizados acessem o módulo de administradores do sistema.
Exibir o número total de denúnciasAtualização: 11/06/2023
RF Descrição
RF07 O sistema deve apresentar o total de denúncias registradas para fornecer uma visão geral do volume de casos relatados.
Exibir o número de denúncias pendentesAtualização: 11/06/2023
RF Descrição
RF08 Deve ser exibido o número de denúncias em espera de atendimento ou análise pelos funcionários da prefeitura.
Exibir o número de denúncias finalizadasAtualização: 11/06/2023
RF Descrição
RF09 O sistema deve mostrar quantas denúncias foram concluídas ou encerradas após tratamento ou ação adequada, indicando o progresso das denúncias e as medidas tomadas.
Exibir o número de denúncias em atendimentoAtualização: 11/06/2023
RF Descrição
RF10 Deve ser apresentado o número de denúncias em processo de análise ou tratamento ativo pelos funcionários responsáveis, fornecendo informações em tempo real sobre as denúncias sendo atendidas.
Exibir o número de denúncias inválidasAtualização: 11/06/2023
RF Descrição
RF11 O sistema deve apresentar a quantidade de denúncias consideradas inválidas ou não relacionadas às arboviroses, auxiliando na filtragem de denúncias não relevantes para o monitoramento.
Exibir um gráfico de denúncias registradas ao longo do anoAtualização: 11/06/2023
RF Descrição
RF12 O sistema deve fornecer um gráfico que ilustre o número de denúncias registradas em cada período ao longo do ano, auxiliando na identificação de possíveis tendências ou sazonalidades nas denúncias.
Exibir um gráfico de denúncias registradas por regiãoAtualização: 11/06/2023
RF Descrição
RF13 O sistema deve apresentar um gráfico que mostre a distribuição geográfica das denúncias, agrupadas por região, permitindo uma análise visual das áreas com maior incidência de arboviroses.
Exibir uma representação gráfica do status das denúnciasAtualização: 11/06/2023
RF Descrição
RF14 O sistema deve fornecer uma visualização gráfica que indique a proporção de denúncias pendentes, finalizadas, em atendimento e inválidas, permitindo entender a distribuição das denúncias de acordo com o status atual.
Listagem de denúnciasAtualização: 11/06/2023
RF Descrição
RF15 O sistema deve listar todas as denuncias cadastradas no sistema e permitir que o usuário pesquise por uma denúncia a partir do CEP, Logradouro ou Bairro.
Visualização de uma denúnciaAtualização: 11/06/2023
RF Descrição
RF16 O sistema deve permitir que o usuário administrador visualize todos os campos de uma denúncia.
