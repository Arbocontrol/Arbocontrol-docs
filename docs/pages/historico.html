Histórico de modificações

Sprint 01

Na data de 12/06/2023, os primeiros requisitos e das regras de negócio do sistema foram definidas. Até então, já existia uma versão de teste do sistema com um painel de controle hospedado em Next.js com uma API em Node.js.

Na data de 22/07, foi decidido que a tecnologia escolhida para desenvolver o backend seria o Node.js.

Construção dos protótipos de telas 23/07/2023 - 29/07/2023 Construção de todas as views de referência.
Construção do formulário de cadastros de denúncias 29/07/2023 - 15/08/2023 Construção do formuário de cadastros de denúncias com base no layout dos protótipos.

Sprint 02

No dia 31/08/2023, ocorreu a primeira reúnião com a equipe da prefeitura para esclarer dúvidas de requisitos de negócios do sistema e compartilhar tudo o que foi produzido até o momento.

Como resultado, foi estabelecido que o sistema continuará sendo divido em dois módulos, um para o público em geral e outro para os profissionais da prefeitura. Além diss, foi proposto que no formulário de cadastro de denuncia será necessário disponibilizar uma nota de versão.
